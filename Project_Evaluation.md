# 프로젝트 고찰 및 기획안 평가 (Project Evaluation & Review)

본 문서는 **PhonemeJP2KR (일본인 학습자를 위한 한국어 발음 교정 서비스)** 개발 완료 시점에서, 프로젝트의 기술적 차별성과 초기 기획안 대비 완성도 및 향후 발전 가능성을 비판적으로 고찰한 문서입니다.

---

## 1. 기술적 차별성과 아키텍처 고찰

순수 ML(Machine Learning) 관점에서 "기존에 학습된 오픈소스 모델(Whisper, Wav2Vec2)과 범용 LLM(Gemini)을 조립한 것뿐이지 않은가?"라는 비판적 질문에 대해, 본 프로젝트는 **'시스템 아키텍처 설계'**와 **'도메인 지식의 결합'** 측면에서 압도적인 차별성을 가집니다.

### ① 듀얼 STT를 활용한 '인지적 착각'의 기술적 분리 (Architecture)
단순히 녹음된 음성을 통째로 LLM에게 던져주고 발음 평가를 요구하는 단순 래퍼(Wrapper) 방식이 아닙니다. 본 프로젝트는 기술적으로 서로 다른 성격을 가진 두 개의 STT 모델을 교차 사용합니다.
- **Whisper**: 언어 모델(LM)이 강력하게 붙어 있어 '원어민의 뇌가 문맥으로 보정해서 듣는 소리(Intelligibility)'를 추출합니다.
- **Wav2Vec2**: 언어 모델 없이 '물리적인 음소(Acoustics)'만 날것으로 추출합니다.

이 **두 모델 출력값의 격차(Gap)를 계산하는 4각 대조 아키텍처**는 일본인 학습자가 겪는 '자신이 내는 소리와 남이 듣는 소리의 괴리'를 기술적으로 정량화해 낸 엄청난 차별점입니다. 단순 프롬프팅이 아니라 멀티 모델 파이프라인(Multi-model Pipeline)을 설계한 결과입니다.

### ② 도메인 지식의 시스템화 (Domain Integration)
범용 AI는 고도의 언어학적 지식을 미리 주입하지 않으면 두루뭉술한 조언만 제공합니다. 
우리는 프롬프트 내부에 **제2언어 습득론(SLA)에 기반한 일본인 특화 에러 패턴(Epenthesis(모음 첨가), Nasalization(비음화 실패) 등)의 룰셋을 하드코딩**했습니다. 이는 범용 AI를 '특정 도메인 전문가 시스템(Expert System)'으로 강제 편입시킨 기술적 제약(Constraining) 기법입니다.

### ③ 오류의 시각화 (HCI: Human-Computer Interaction)
외국어 학습의 가장 큰 장벽인 **'음운론적 귀머거리(Phonological Deafness)'** 현상을 해결하기 위해, Wav2Vec2의 물리적 소리를 **일본어 가타카나로 강제 매핑하여 시각화**했습니다. 
"당신의 뇌는 지금 이렇게 인식하고 있다"를 화면에 UI로 박아버린 것은 기술적 최적화를 넘어선 뛰어난 UX/HCI 설계입니다.

## 2. 초기 기획안 대비 완성본 평가

초기 기획안(PDF)의 탄탄한 시장 조사와 기술적 타당성을 바탕으로, 현재 완성된 결과물은 기획안의 한계를 뛰어넘어 진화했습니다.

### 🏆 기획안보다 '압도적으로' 진화한 부분

1. **단순 비교를 넘어선 '가타카나 시각화(L1 간섭)'의 도입**
   - **기획안**: Whisper(의도)와 Wav2Vec2(실제)의 텍스트 차이를 비교하여 오류를 탐지하는 '기계적/규칙 기반 비교' 수준에 머물러 있었습니다.
   - **완성본**: 단순히 틀린 글자를 찾는 것을 넘어, '왜 틀렸는지'를 일본어 모라(Mora) 박자 시스템의 관점에서 **가타카나로 역치환하여 시각화**했습니다. 교육학적 가치를 극대화한 신의 한 수입니다.

2. **LLM(Gemini)을 활용한 '교수급 코칭 피드백'의 자동화**
   - **기획안**: 규칙 기반 모델(Rule-based)을 구현하여 음소 오류를 강제로 맵핑하려는 다소 딱딱한 엔지니어링 계획(형태소 분석기 Mecab 적용 등)이 잡혀 있었습니다.
   - **완성본**: 복잡한 하드코딩 대신 최신 LLM(Gemini 2.5 Flash)에 SLA 프롬프트를 주입하여, **살아있는 원어민 선생님처럼 유연하고 친절한 맞춤형 코칭**을 일본어로 제공하게 되었습니다.

### 💡 향후 버전을 위해 원본 기획안에서 '더 건져올 수 있는' 아이디어

기획안에는 추후 고도화나 사이드 프로젝트(KR2JP) 시 반드시 도입해야 할 훌륭한 아이디어들이 남아 있습니다.

1. **K-콘텐츠(드라마/애니메이션) 대사 섀도잉 (기획안 6번 항목)**
   - **현재**: 사용자가 빈칸에 타겟 문장을 직접 입력해야 합니다.
   - **발전 방향**: 초기 화면에 유명 K-드라마 명대사 리스트를 띄워주고, 이를 클릭하면 바로 타겟 문장으로 세팅되어 섀도잉 연습을 할 수 있게 만들면 유저 리텐션(재방문율)을 크게 높일 수 있습니다.

2. **`langdetect`를 활용한 한/일 혼용 구간(Code-Switching) 자동 분리**
   - **현재**: 오로지 한국어만 녹음된다고 가정하고 분석합니다.
   - **발전 방향**: 기획안에 명시된 대로 `langdetect`를 이용해 일본어 발화는 무시하고 한국어 발화 구간만 추출해 점수를 매기게 한다면, '유튜브 브이로그 영상 기반 트레이닝 서비스'로 확장이 가능합니다.

3. **커스텀 데이터 파인튜닝 (기획안 5번 항목)**
   - **현재**: 범용 한국어 Wav2Vec2 모델(`kresnik`)을 사용 중입니다.
   - **발전 방향**: '일본인이 어설프게 발음한 한국어 음성' 데이터셋을 모아 모델을 파인튜닝(Fine-tuning)하면, 일본인 특유의 악센트와 오류를 잡아내는 독보적인 상용화 모델을 구축할 수 있습니다.
